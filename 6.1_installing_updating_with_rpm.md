# RPM 
- Managing softwares in the form of rpm is easier and simpler then working with software extracted from archieve
- With rpm packages, admins can check which files get installed, removed and supporting packages are present
- local rpm database - stores information about installed packages
- ALl packages are provied by redhat are as rpms

# rpm naming convention 
- <packagename>.<version>.<release>.<processor_architecture_x86_64>

# Repository 
- rpm packages are downloaded from repository
- Repository is central location for storing and managing the rpm software packaages

# rpmn package contains
- files the pacakge will install
- metadata about package such as name, version, release, architecture
- scripts that run when we install, delete, update package

# GNU privacy guard
- All the rpm packages are digitally signed by providers using GPG keys
- REDHAT also digitall signs the rpm packages
- rpm system will verify if the packages are signed by appropriate keys

# Updating package
- Only 1 version of the package can exist
- It will remove old version and installs new version
- The config files remains

# rpm utility 
Tool used to get detailed information about the contents of rpm package
##  List all  installed packages - rpm -qa

## Which package provides file - rpm -qf <filename>
