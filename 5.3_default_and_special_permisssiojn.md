# setuid
- File :- When we execute a file. The file executes with the permissions of owner of the file and not with user who executed the file
- Dir :- No affect
## To set
```
chmod u+s a.sh
chmod 4755 a.sh
```

```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# ll /bin/cat
-rwxr-xr-x. 1 root root 40760 Nov 26  2024 /bin/cat

[devops@ceph-vipin-tfa-q5w08s-node6 ~]$
[devops@ceph-vipin-tfa-q5w08s-node6 ~]$ ll /etc/shadow
----------. 1 root root 1896 Apr 12 12:57 /etc/shadow
```
user cannot directly access as executable will run as devops user and he does not have permisssion to execute it
```
[devops@ceph-vipin-tfa-q5w08s-node6 ~]$ cat /etc/shadow

# Setgid
- When we create a file inside a folder by default the creator user and creator group is added for the file
- We can make the dir owning group to be added as owning group of the files by default using setgid.
- The owning group is added by default as owning group for all files and directories created under the folder
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# chmod g+s test
[root@ceph-vipin-tfa-q5w08s-node6 ~]# ls -ld test
drwxr-sr-x. 2 devops developers 21 Apr 12 13:03 test
[root@ceph-vipin-tfa-q5w08s-node6 ~]# cd test
[root@ceph-vipin-tfa-q5w08s-node6 test]# touch new
[root@ceph-vipin-tfa-q5w08s-node6 test]# ll
total 0
-rw-r--r--. 1 root developers 0 Apr 12 13:04 new
-rw-r--r--. 1 root root       0 Apr 12 13:03 test123
```
