# su  - substitute
- substitue the user to another user
- We need to provide other users password
- su - devops --> login as devops user
- su - ---> login as root user

# sudo - super user do 
- we need to provide our own password
- controlled access
- only if the user is part of /etc/sudoers or /etc/sudoers.d they can use sudo commands
- Recommended approach
- Should never modify /etc/sudoers file
- %wheel ALL=(ALL) NOPASSWORD: ALL
