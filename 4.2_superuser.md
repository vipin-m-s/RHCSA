# su  - substitute
- substitue the user to another user
- We need to provide other users password
- su - devops --> login as devops user
- su - ---> login as root user
- when root user executed su he does not need to provide the password

## su - 
- enters a interactive login shell, where the env is switched to target users env

## su 
- enters a interactive login shell as user u r trying to switchto , paritally uses the current users env, 

# sudo - super user do 
- we need to provide our own password
- controlled access
- only if the user is part of /etc/sudoers or /etc/sudoers.d they can use sudo commands
- Recommended approach
- Should never modify /etc/sudoers file
- %wheel ALL=(ALL) NOPASSWORD: ALL
- entered passowrd is cached for 5 minutes. sudo -k to clear cache. If NOPASSWORD is added then sudo can execute without password

## In sudoers file
- to grant access to user - nick ALL=(ALL) ALL
- to grant access to group - %QA ALL=(ALL) ALL
- content in any file in /etc/sudoers.d will also be included in /etc/sudoers file. 

# RHEL 10 does not have a valid root password by default. we cannot login as root

# Using sudo su cephuser
- id is updated to cephuser
- $HOME is updated to cephuser
- $PATH is updated to cephuser
- $USER is updated to cephsuer
- pwd - current working directory remains as is
- the roots .bashrc configurations are not loaded. Does not give a  fresh root login shell
