# useradd
## WHen creating users, the below files are referenced
- defualt configs - /etc/default/useradd
```
[cephuser@ceph-vipin-tfa-q5w08s-node6 ~]$ cat /etc/default/useradd
# useradd defaults file
GROUP=100
HOME=/home
INACTIVE=-1
EXPIRE=
SHELL=/bin/bash
SKEL=/etc/skel
CREATE_MAIL_SPOOL=yes
```
- /etc/log.defs has password expirty, min length , user id etc 
- /etc/skel - skeleton  What should be the home directory of the newly created user have ?

# Adding user 
- -c description
- -s the shell for user
- -u user id
- -d the home directory 
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# useradd -c "Test user 123" -s /sbin/nologin -u 10000 -d /dev/null testuser123
useradd: warning: the home directory /dev/null already exists.
useradd: Not copying any file from skel directory into it.

[root@ceph-vipin-tfa-q5w08s-node6 ~]# cat /etc/passwd | grep testuser
testuser:x:1001:1001::/home/testuser:/bin/bash
testuser123:x:10000:10000:Test user 123:/dev/null:/sbin/nologin
```

# modifying user using usermod

```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# grep cephuser /etc/passwd

```

# delting a user
- home directory still remains even if user is deleted
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# userdel testuser
[root@ceph-vipin-tfa-q5w08s-node6 ~]# ls -l /home
total 0
drwx------. 4 cephuser cephuser 110 Apr 11 11:53 cephuser
drwx------. 5     1001     1001 120 Apr 12 02:32 testuser
[root@ceph-vipin-tfa-q5w08s-node6 ~]#
```
The user is deleted form etc/passwd and /etc/group
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# grep 1001 /etc/passwd
[root@ceph-vipin-tfa-q5w08s-node6 ~]# ^passwd^group
grep 1001 /etc/group
[root@ceph-vipin-tfa-q5w08s-node6 ~]#
```

# deleting usser and his home directory

```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# useradd mo
```

# When a new user is created he will get the same ID as previous user and the home directory of deleted user is now the new users
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# useradd mo
[root@ceph-vipin-tfa-q5w08s-node6 ~]# ll /home
total 0
drwx------. 4 cephuser cephuser 110 Apr 11 11:53 cephuser
drwx------. 5 mo       mo       120 Apr 12 04:30 mo
drwx------. 5     1001     1001 120 Apr 12 02:32 testuser
[root@ceph-vipin-tfa-q5w08s-node6 ~]# userdel mo
```
After deleting mo the folder belongs to user id 10001 and group 10001 
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# ll /home
total 0
drwx------. 4 cephuser cephuser 110 Apr 11 11:53 cephuser
drwx------. 5    10001    10001 120 Apr 12 04:30 mo
drwx------. 5     1001     1001 120 Apr 12 02:32 testuser
```
add new user mo2. He will now get the id of 10001 and group 10001 and hence the fodler mo is now mo2's.
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# useradd mo2
[root@ceph-vipin-tfa-q5w08s-node6 ~]# ll /home
total 0
drwx------. 4 cephuser cephuser 110 Apr 11 11:53 cephuser
drwx------. 5 mo2      mo2      120 Apr 12 04:30 mo
drwx------. 5 mo2      mo2      120 Apr 12 04:30 mo2
drwx------. 5     1001     1001 120 Apr 12 02:32 testuser
```

# Locking user. 
- it is not recommended to delete user
- It is recommended to use identity provider IDM, active directory for user management
- We cna also lock an user, they will not be able to login
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# usermod -L mo2
```
