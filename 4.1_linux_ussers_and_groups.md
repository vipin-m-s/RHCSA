# types of user
- superuser - has full system access, administrator
- system user - used by processes, they would not be given superuser access, this is not used by users
- regualr user - for regular users day to day activites. has limited access

# cat /etc/passwd
contins the ifnormation about user
```
[root@ceph-vipin-tfa-q5w08s-node6 3.1]# grep cephuser /etc/passwd
cephuser:x:1000:1000::/home/cephuser:/bin/bash
```
1 -> username
2 -> earlier password was stored
3 -> user id 
4 -> primary group id 
5 -> can store anything , user full name
6 -> home directory 
7 -> default shell of user, we can set to /sbin/nologin shell will disallow interactive logins 

# cat /etc/group 
copntinas info about group 
```
[root@ceph-vipin-tfa-q5w08s-node6 3.1]# grep cephuser /etc/group
cephuser:x:1000:

yggdrasil-worker:x:998:yggdrasil

```
1 -> group name
2 -> password of the group , if you knew password of the group, we can add ourself to that group 
3 -> group id
4 -> members of the group, the group specific to the user does not need membership

# primary group and supplementary group 
- every user has primary group and supplementary group
- when user is created a group with same name is created whcich is priomary group by user
- the primary group owns the file
- We can then add other grouops
- If user has user1, user2, user3 group membership, he can  read files for all the groups.
