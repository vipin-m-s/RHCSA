- RHEL uses extents file system XFS as default local file system
- RHEL supports ext4 file system for local files
- IN RHEL 10 supports exFAT for removable media

# Mounting file system 
- Mounting - making the file system available for use 
- Mount point - the directory to which we mount the file system
- Most file systems are automatically mounted when we boot

# using block devices
- block device - a file that provides low level access to block device storage
- HDD or SSD Block device is paritioned into smaller block devices
- These paritions must be formatted with file system before the file system can be mounted
- /dev folder will hold the block devices and rhel creates it automatically

- block device naming convention
  - for sata, sas, scsi and usb - /dev/sda, /dev/sdb
  - virtio-blk - /dev/vda, /dev/vdb
  - virtio-sata - /dev/sda
  - sd, mmc , emmc - mmcblk0

# Paritions
- storage device is divided into multiple paritions
- Each paritition should be formatted with file system
- Each parition can be used for different purpose
- One for home directory, one for logs etc.
- for sata - /dev/sda1, /dev/sda2
- for virtualized - /dev/vda, /dev/vda2
- for nvme - /dev/nvme0n1p1
  - nvmeX - device
  - nY - namespace
  - pZ - parition
  - /dev/nvme0n1p1
