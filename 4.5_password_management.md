# Password info can be found in /etc/shadow 
- only root users can view the file
- Columns:-
  1) username
  2) hashed password ( $y - yes encrypt , $6 - sha512 )
  - first part is algorithm used
  - second is options
  - third is salt
  - fourth is password
  3) Days since the EPOCH when the password was changed
  4) Min days they can update password
  5) Max days before password expires
  6) 7 days before expiry they will get warning
  7) How long can they use account after password expires
  8) How long before account expires
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# grep devops /etc/shadow
devops:$y$j9T$HjmYqrD279eMq.FeVRL.B.$IoC.VMoM61dx/oFLOb.OS0o40S4NG8Io3cFkTDXfVfB:20555:0:99999:7:::
```

# We can change the password configs using chage - change aging of password
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# chage -m 2 -M 30 -W 10 -I 5 -E 2026-12-31 devops
[root@ceph-vipin-tfa-q5w08s-node6 ~]# grep devops /etc/shadow
devops:$y$j9T$HjmYqrD279eMq.FeVRL.B.$IoC.VMoM61dx/oFLOb.OS0o40S4NG8Io3cFkTDXfVfB:20555:2:30:10:5:20818:
```

# We can also lock a password
- Once user is locked he cannot login 
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# usermod -L devops
[root@ceph-vipin-tfa-q5w08s-node6 ~]# ssh devops@localhost
devops@localhost's password:
Permission denied, please try again.
```

# Locked user will have ! before his hashed passwordf
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# grep devops /etc/shadow
devops:!$y$j9T$HjmYqrD279eMq.FeVRL.B.$IoC.VMoM61dx/oFLOb.OS0o40S4NG8Io3cFkTDXfVfB:20555:2:30:10:5:20818:
```


# We can check status of password using passwd -S devops
- L locked
- P active
- aging information
```
[root@ceph-vipin-tfa-q5w08s-node6 ~]# passwd -S devops
devops L 2026-04-12 2 30 10 5
```

# Disadvantage of locking user - If ssh key is configured user can still login using ssh keys
- lock the account and expire the account for former employees
```
usermod -L -e 2026-1-1 devops
```
- to unlock and remove expiration
```
usermod -U -e '' devops
```
